## Directory Purposes

**`lang/`:**
- Purpose: Localization strings consumed by the CSS `Localizer` at runtime
- Contains: One flat JSON file per supported locale; keys use dot-notation namespacing
- Key files: `lang/en.json` is the canonical source of all message keys
- To add a locale: create `lang/{IETF-code}.json` mirroring every key in `lang/en.json`
- To add a message: add the key to every existing `lang/*.json` file

**`src/ReservedSlots/`:**
- Purpose: The .NET 8 class library project that compiles to the plugin DLL
- Contains: One `.cs` source file holding all types; one `.csproj` project file
- Key file: `src/ReservedSlots/ReservedSlots.cs` — config class, plugin class, enums, all methods

**`src/ReservedSlots/bin/Debug/net8.0/`:**
- Purpose: Build output; contains the deployable plugin artifacts
- Generated: Yes — produced by `dotnet build`
- Committed: Yes (currently tracked; used for release distribution)
- Deploy by copying this directory's contents to `csgo/addons/counterstrikesharp/plugins/ReservedSlots/` on the game server

**`src/ReservedSlots/obj/`:**
- Purpose: MSBuild intermediate cache files
- Generated: Yes
- Committed: Partially (some cached files present in repo; ideally excluded)

**`.planning/codebase/`:**
- Purpose: GSD planning and analysis documents
- Generated: No
- Committed: Yes

## Naming Conventions

**Files:**
- C# source file matches class/namespace name: `ReservedSlots.cs`
- Project file matches assembly name: `ReservedSlots.csproj`
- Lang files use IETF locale codes: `en.json`, `lv.json`, `pt-BR.json`

**Types:**
- Plugin class: PascalCase, matches namespace and assembly: `ReservedSlots`
- Config class: PascalCase with `Config` suffix: `ReservedSlotsConfig`
- Enums: PascalCase: `KickType`, `KickReason`, `ReservedType`
- Enum values: PascalCase: `Random`, `HighestPing`, `ServerIsFull`, `VIP`, `Admin`, `None`

**Members:**
- Public config properties: camelCase with `[JsonPropertyName]` for the JSON key (e.g., `reservedFlags`)
- Private cached nullable fields: camelCase with `_` prefix (e.g., `_cachedVisibleMaxPlayers`)
- State collection fields: camelCase, no prefix (e.g., `waitingForKick`, `reservedPlayers`)
- Public/internal methods: PascalCase (e.g., `PerformKick`, `SetKickImmunity`)
- Private methods: mixed — most are PascalCase; `getPlayerToKick` is camelCase (legacy inconsistency)

**Lang keys:**
- Pattern: `{Surface}.{EventOrState}[.{Detail}]`
- Examples: `Hud.ServerIsFull`, `Chat.PlayerWasKicked.ReservedPlayerJoined`

## Module Boundaries

This is a single-module, single-file project. All types are in the `ReservedSlots` namespace:

| Type | Role |
|------|------|
| `ReservedSlotsConfig` | Data container; deserialized by CSS framework |
| `ReservedSlots` | Plugin logic; all methods are instance methods on this class |
| `KickType` (enum) | Enumerates victim selection strategies |
| `KickReason` (enum) | Distinguishes why a kick was triggered (affects message and delay) |
| `ReservedType` (enum) | Player tier: `Admin`, `VIP`, or `None` |

## Where to Add New Code

**New config option:**
1. Add a property to `ReservedSlotsConfig` in `src/ReservedSlots/ReservedSlots.cs`
2. Add `[JsonPropertyName("Human Readable Key")]` attribute
3. Add validation in `OnConfigParsed` if the value is constrained
4. Document in `README.md`

**New kick selection strategy:**
1. Add value to `KickType` enum
2. Add `case (int)KickType.YourValue:` to `getPlayerToKick()` switch
3. Update `README.md` Kick Type table

**New reserved slot method:**
1. Add `case N:` to `switch (Config.reservedSlotsMethod)` in `OnPlayerConnect`
2. For non-trivial logic, extract to a private `Handle*ModeJoin()` method (see `HandleOverflowModeJoin` as the pattern)
3. Update validation range in `OnConfigParsed`
4. Update `README.md`

**New localization string:**
1. Add key to `lang/en.json`
2. Add the same key to `lang/lv.json` and `lang/pt-BR.json`
3. Reference via `Localizer["Key.Name"]` or `Localizer["Key.Name", arg]` in code

**New supported locale:**
1. Create `lang/{locale-code}.json`
2. Copy all keys from `lang/en.json` and translate values
